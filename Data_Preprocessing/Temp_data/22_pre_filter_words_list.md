### 脊柱多模态数据集语义筛选清单（白名单与黑名单）

#### 一、 脊柱白名单（White List）
**匹配逻辑**：采用“动态多维字段交叉验证”机制。对于 CT 和 X光 模态，考察三个核心字段（部位 `BodyPartExamined`、序列描述 `SeriesDescription`、检查描述 `StudyDescription`）；对于 MRI 模态，增加第四个硬件字段（接收线圈名称 `ReceiveCoilName`）。只要目标字段中包含以下任意字符串（忽略大小写），即判定为有效骨科/脊柱数据。

| 关键词集合 | 类别 | 保留理由与临床解析 |
| :--- | :--- | :--- |
| `SPINE`, `LSPINE`, `TSPINE`, `CSPINE`, `L-S`, `T-L`, `L-SPINE`, `T-SPINE`, `C-SPINE`, `T6-L3` | 解剖学英文 | 标准的脊柱及各节段（颈、胸、腰、骶）英文缩写，直接证明扫描目标。 |
| `椎`, `ZHUI`, `ji zhu` | 解剖学中文/拼音 | 国内医院 PACS 系统特有的拼音或中文输入，精准定位脊柱扫描。 |
| `BONE` | CT重建算法 | 骨窗重建算法。专为观察骨松质、骨皮质细节生成，是骨折检测的绝对核心数据。 |
| `STIR`, `DIXON` | MR压脂序列 | 短T1反转恢复及水脂分离序列。能有效抑制脂肪信号，突出骨髓水肿与隐匿性骨折。 |
| `SAG`, `COR` | 空间切面 | 矢状面与冠状面。脊柱成像最依赖纵向切面以观察椎体序列和椎间盘，横断面（Axial）单独存在价值较低。 |
| `SPINEARRAY`, `AIM_SPINE`, `CTSP_ADAPTER` | MR物理线圈 | 物理约束特征。只要使用了脊柱专用接收线圈或适配器，成像区域必然是脊柱。 |
| `位` | X光投照体位 | X光特有后缀。通常为“正侧位”、“斜位”的截断残留，由于X光全盘为骨科，属于防误杀捞回特征。 |
| `3D_5mm COR_1`, `3D_Batch`, `3D_3mm SAR_1`, `3D_3mm SAR_2` | 3D后处理集 | 工作站基于容积扫描生成的批量三维重组序列（通常是冠状/矢状面打包）。 |
| `1.25mm std/stnd`, `5mm std/stnd`, `1.5 x 1.0`, `1.5 x 1.2` | 物理层厚与重建参数 | 经人工抽查确认，这些在数据集内主要指代含有脊柱的高分辨率薄层或标准重组扫描，需进行捞回。 |


#### 二、 脊柱黑名单（Black List）
**匹配逻辑**：在未命中白名单的情况下，或者作为白名单的前置排除条件，包含以下特征的数据将被彻底物理隔离。

| 关键词集合 | 类别 | 剔除理由与临床解析 |
| :--- | :--- | :--- |
| `LUNG` | CT非骨科算法 | 肺窗重建算法。此类图像骨骼严重过曝，骨皮质细节完全丢失，会严重干扰骨科模型特征提取。 |
| `HEAD`, `BRAIN`, `CHEST` | 非骨科解剖部位 | 纯颅脑或纯胸部平扫。在没有命中白名单骨科参数的情况下，属于无骨科诊断价值的废片。 |
| `DOSE REPORT` | 行政记录截图 | 辐射剂量报告。系统自动生成的包含表格文字的 2D 截图，无 3D 医学影像矩阵。 |
| `SCREEN SAVE` | 屏幕截图 | 技师操作界面的屏幕捕获，缺乏医学诊断价值和原始空间坐标。 |
| `PROCESSED IMAGES`, `3D_DEFAULT_1` | 破坏性/非标准后处理 | 经过工作站非标准人工拉伸或重组的次生图像，破坏了原始物理空间的一致性。 |
| `3D_CT_VR_VOLREN_COLLECTION_1` | 容积渲染截图 | VR容积渲染彩色表面截图，本质为供人眼观看的 2D 照片，无法输入 3D 网络。 |
| `SCOUT`, `2D` | 定位图与低质图 | 扫描前的快速低清定位图（Localizer）或 2D 轮廓图，层厚大、分辨率极低，无 3D 重建价值。 |
| `tj tj` | 体检杂项 | 普遍代指常规体检项目，缺乏骨科针对性协议，画质和切面通常无法满足骨科要求。 |
| `0915013854`, `CTN0000049` | 误填流水号/设备号 | 技师将患者流水号或设备机身号误填入检查描述中，缺乏解剖学依据，作为废片处理。 |
| `EMPTY_VALUE`, `NAN` | 缺失值与盲盒 | 在关键语义字段全空或系统读取为空值的条件下，采取“宁可错杀不可放过”原则，避免引入不可控的脏数据干扰。 |


**实际筛选只要目标字段（CT/X光为3个，MRI为4个）的值都不包含白名单字段即筛除，不用再跑一遍黑名单了。**
